Computer Architecture: Binary Numbers & Data Types

Understanding how computers represent and manipulate data at the most fundamental level

What is Computer Architecture?

Computer Architecture is the study of how computers are designed and how they work at a fundamental level.

It covers:

  • How data is represented inside a computer
  • How instructions are executed
  • How memory and registers are organized
  • How the CPU processes binary numbers

Binary System & Bits

Computers are electronic machines that work by controlling the flow of electrons. They recognize two fundamental states:

  • Presence of voltage - represented as state "1"
  • Absence of voltage - represented as state "0"

This binary system is simpler to implement than measuring or regulating precise voltage levels.

Bits & Binary Digits

The basic unit of information in computing is the binary digit, or bit.

Multiple bits can represent more complex information:

  • 2 bits = 4 possible states: 00, 01, 10, 11
  • 3 bits = 8 possible states: 000, 001, 010, 011, 100, 101, 110, 111
  • n bits = 2n possible states
0 1 0 1 1 0 1 0

Number Systems in Computers

Computers don't understand human languages like English. They only understand binary digits (bits):

0 → Low voltage (off)
1 → High voltage (on)

Other Number Systems:

  • Decimal (Base 10) – what humans use (0–9)
  • Binary (Base 2) – what computers use (0, 1)
  • Hexadecimal (Base 16) – often used as shorthand in computing (0–9, A–F)

Example Conversions:

Decimal Binary Hexadecimal
10 1010 A
255 11111111 FF
42 101010 2A

Data Representation

Computers need to represent various types of information:

  • Numbers: Signed, unsigned, integers, floating point
  • Text: Characters, strings
  • Images: Pixels, colors, shapes
  • Sound: Audio waveforms
  • Logical values: True, false
  • Instructions: Machine code

A data type defines:

  • How data is represented in the computer
  • What operations can be performed on the data

Different data types require different representation schemes and operations.

Unsigned Integers

Unsigned numbers = only positive numbers (including zero).

Range for n bits = 0 to (2ⁿ − 1)

Examples:

4-bit Unsigned

Range: 0 to 15
Example: 1111 = 15

8-bit Unsigned

Range: 0 to 255
Example: 11111111 = 255

00000001 = 1
11111111 = 255
01010101 = 85

Signed Integers

Signed numbers = positive and negative numbers.

The Most Significant Bit (MSB) is used as the sign bit:

0 → Positive
1 → Negative

Ways to Represent Signed Numbers:

  • Sign-magnitude: MSB indicates sign (0=positive, 1=negative)
  • One's complement: Flip all bits to represent negative
  • Two's complement: Most widely used in modern computers

Two's complement simplifies arithmetic operations and avoids having two representations of zero.

Two's Complement

To represent a negative number in two's complement:

  1. Start with the positive number
  2. Flip every bit (one's complement)
  3. Add 1 to the result
5 = 00101
One's complement = 11010
Add 1 = 11011 (-5)

Shortcut: Copy bits from right until first 1, then flip remaining bits.

Integer Range

Range of an n-bit two's complement number:

  • Minimum: -2n-1
  • Maximum: 2n-1 - 1

For example, an 8-bit two's complement number ranges from -128 to 127.

The most negative number (-2n-1) has no positive counterpart.

Arithmetic in Two's Complement

Two's complement addition is straightforward binary addition:

  • Add from right to left, propagating carry
  • Ignore final carry out

Subtraction is performed by negating the subtrahend and adding:

104 - 16 = 104 + (-16) = 88

Sign Extension

To add numbers with different bit lengths:

  • Replicate the MSB (sign bit) to fill the new bits
  • For positive numbers: fill with 0s
  • For negative numbers: fill with 1s
4 (0100) → 8-bit: 00000100
-4 (1100) → 8-bit: 11111100

Overflow

Overflow occurs when the result of an operation cannot be represented in the available bits:

  • Signs of operands are the same, but sign of result is different
  • Carry into MS bit ≠ carry out of MS bit
8 + 9 = 17 (overflow in 5-bit)
01000 + 01001 = 10001 (-15)

Parity Check (Error Detection)

Computers must ensure data is not corrupted during transmission. Parity bits are extra bits added for error detection.

Types of Parity

Even Parity

Total number of 1s (data + parity) is even

Odd Parity

Total number of 1s is odd

Example: Data = 1010 (two 1s)
Even parity → add 0 → 10100
Odd parity → add 1 → 10101

Parity can detect single-bit errors, but not correct them.

LC-3 (Little Computer 3) Basics

The LC-3 is a simple educational computer architecture used to teach how CPUs work.

Key Features:

  • 16-bit architecture (instructions and data are 16 bits)
  • Registers: 8 general-purpose registers (R0–R7)
  • Memory: 65,536 locations (2¹⁶)
  • Instruction Set: Load, Store, Add, AND, NOT, Branch, etc.
Example LC-3 instruction: ADD R1, R2, R3
Meaning: R1 ← R2 + R3

LC-3 Data Types:

  • Only one supported data type: 16-bit 2's complement signed integer
  • Operations: ADD, AND, NOT
  • Other data types supported by interpreting 16-bit values as logical, text, fixed-point, etc.

Logical Operations

Operations on logical TRUE or FALSE with two states. Takes one bit to represent: TRUE=1, FALSE=0

View n-bit number as a collection of n logical values with operation applied to each bit independently.

Truth Tables:

Operation Description Use Case
AND Output 1 only if both inputs are 1 Clearing bits (AND with 0)
OR Output 1 if at least one input is 1 Setting bits (OR with 1)
NOT Invert the input bit Complementing values

Practical Applications:

AND - Clearing Bits:
11000101 AND 00001111 = 00000101
AND with zero = 0, AND with one = no change

OR - Setting Bits:
11000101 OR 00001111 = 11001111
OR with zero = no change, OR with one = 1

NOT - Flipping Bits:
NOT 11000101 = 00111010
Unary operation -- flips every bit

Hexadecimal Notation

It is often convenient to write binary (base-2) numbers as hexadecimal (base-16) numbers instead.

  • Fewer digits — four bits per hex digit
  • Less error prone — easy to corrupt long string of 1's and 0's

Conversion Table:

Binary Hex Decimal Binary Hex Decimal
0000 0 0 1000 8 8
0001 1 1 1001 9 9
0010 2 2 1010 A 10
0011 3 3 1011 B 11
0100 4 4 1100 C 12
0101 5 5 1101 D 13
0110 6 6 1110 E 14
0111 7 7 1111 F 15

Converting from Binary to Hexadecimal:

Every four bits is a hex digit. Start grouping from right-hand side:

011101010001111010011010111
Group: 0111 0101 0001 1110 1001 1010 111
Result: 751E9A7 = 7515E9A7

This is not a new machine representation, just a convenient way to write the number.

Floating-Point Numbers

For very large and very small numbers, we use floating-point representation.

Large values: 6.023 × 10²³ — requires 79 bits
Small values: 6.626 × 10⁻³⁴ — requires >110 bits

Use equivalent of "scientific notation": F × 2^E

Need to represent F (fraction), E (exponent), and sign.

Fixed-Point Representation

Fractions can be represented using a "binary point" to separate positive and negative powers of two:

00101000.101 = 40.625
25 + 23 + 2-1 + 2-3

Standard addition and subtraction operations work as long as binary points are aligned.

IEEE 754 Floating-Point Standard (32-bits):

Sign (S) Exponent Fraction
1 bit 8 bits 23 bits
Value = (-1)^S × (1 + fraction) × 2^(exponent - 127)

Example:

Single-precision: 10111111010000000000000000000000

• Sign is 1 – number is negative
• Exponent field is 01111110 = 126 (decimal)
• Fraction is 0.100000000000… = 0.5 (decimal)

Value = -1.5 × 2^(126-127) = -1.5 × 2^(-1) = -0.75

Text Representation & ASCII

ASCII: Maps 128 characters to 7-bit code, including both printable and non-printable characters.

Interesting Properties of ASCII:

  • Decimal digits ('0', '1', …): ASCII code = decimal value + 48
  • Case conversion: Difference between uppercase and lowercase is exactly 32
  • Alphabetical order: ASCII values maintain alphabetical ordering
Examples:
'0' = 48, '1' = 49, '9' = 57
'A' = 65, 'a' = 97 (difference = 32)
'Z' = 90, 'z' = 122

Question: Are 128 characters enough? Modern systems use Unicode for international character support!

Other Data Types

Common data types beyond numbers and text:

  • Text strings: Sequences of characters, often null-terminated
  • Images: Arrays of pixels (monochrome: 1 bit/pixel; color: RGB components)
  • Sound: Sequences of fixed-point numbers representing audio waveforms

These are typically implemented in software rather than directly in hardware.

Key Takeaways & Summary

Essential Concepts:

  • Binary Foundation: Computers use binary (0 and 1) for all operations and data storage.
  • Number Types: Unsigned (positive only) vs Signed (positive & negative using two's complement).
  • Operations: Logical operations (AND, OR, NOT) and arithmetic work at the bit level.
  • Error Detection: Parity bits help detect transmission errors in digital systems.
  • LC-3 Architecture: Simple 16-bit educational computer for learning CPU fundamentals.
  • Data Types: Various representations: integers, floating-point, text, and multimedia data.

Real-World Impact:

Computer architecture and data representation are the foundation of:

  • Digital Electronics: Every chip in your computer
  • Programming: Data types and memory management
  • Database Systems: Data storage and retrieval
  • Network Communications: Data transmission protocols
  • AI/ML: Neural network computations
  • Graphics & Multimedia: Image and sound processing
