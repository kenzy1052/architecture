Binary System & Bits

Binary States

Computers are electronic machines that work by controlling the flow of electrons. They recognize two fundamental states:

  • Presence of voltage - represented as state "1"
  • Absence of voltage - represented as state "0"

This binary system is simpler to implement than measuring or regulating precise voltage levels.

Bits & Binary Digits

The basic unit of information in computing is the binary digit, or bit.

Multiple bits can represent more complex information:

  • 2 bits = 4 possible states: 00, 01, 10, 11
  • 3 bits = 8 possible states: 000, 001, 010, 011, 100, 101, 110, 111
  • n bits = 2n possible states

Digital System

A computer is a binary digital system:

  • Binary: Has only two states (0 and 1)
  • Digital: Uses a finite number of symbols
0 1 0 1 1 0 1 0

Data Representation

Types of Data

Computers need to represent various types of information:

  • Numbers: Signed, unsigned, integers, floating point
  • Text: Characters, strings
  • Images: Pixels, colors, shapes
  • Sound: Audio waveforms
  • Logical values: True, false
  • Instructions: Machine code

Data Types

A data type defines:

  • How data is represented in the computer
  • What operations can be performed on the data

Different data types require different representation schemes and operations.

Integer Representation

Unsigned Integers

Weighted positional notation is used to represent unsigned integers:

329 = 3×100 + 2×10 + 9×1
101 = 1×4 + 0×2 + 1×1 = 5

An n-bit unsigned integer represents values from 0 to 2n-1.

Signed Integers

Three common representations for signed integers:

  • Sign-magnitude: MSB indicates sign (0=positive, 1=negative)
  • One's complement: Flip all bits to represent negative
  • Two's complement: Most widely used in modern computers

Two's complement simplifies arithmetic operations and avoids having two representations of zero.

Two's Complement

To represent a negative number in two's complement:

  1. Start with the positive number
  2. Flip every bit (one's complement)
  3. Add 1 to the result
5 = 00101
One's complement = 11010
Add 1 = 11011 (-5)

Shortcut: Copy bits from right until first 1, then flip remaining bits.

Integer Range

Range of an n-bit two's complement number:

  • Minimum: -2n-1
  • Maximum: 2n-1 - 1

For example, an 8-bit two's complement number ranges from -128 to 127.

The most negative number (-2n-1) has no positive counterpart.

Operations

Addition & Subtraction

Two's complement addition is straightforward binary addition:

  • Add from right to left, propagating carry
  • Ignore final carry out

Subtraction is performed by negating the subtrahend and adding:

104 - 16 = 104 + (-16) = 88

Sign Extension

To add numbers with different bit lengths:

  • Replicate the MSB (sign bit) to fill the new bits
  • For positive numbers: fill with 0s
  • For negative numbers: fill with 1s
4 (0100) → 8-bit: 00000100
-4 (1100) → 8-bit: 11111100

Overflow

Overflow occurs when the result of an operation cannot be represented in the available bits:

  • Signs of operands are the same, but sign of result is different
  • Carry into MS bit ≠ carry out of MS bit
8 + 9 = 17 (overflow in 5-bit)
01000 + 01001 = 10001 (-15)

Logical Operations

Basic logical operations applied bit by bit:

Operation Description Use Case
AND Output 1 only if both inputs are 1 Clearing bits (AND with 0)
OR Output 1 if at least one input is 1 Setting bits (OR with 1)
NOT Invert the input bit Complementing values

Floating Point & Fractions

Fixed-Point Representation

Fractions can be represented using a "binary point" to separate positive and negative powers of two:

00101000.101 = 40.625
25 + 23 + 2-1 + 2-3

Standard addition and subtraction operations work as long as binary points are aligned.

Floating-Point Representation

For very large or very small numbers, use scientific notation: F × 2E

IEEE 754 Floating-Point Standard (32 bits):

  • 1 bit: Sign (S)
  • 8 bits: Exponent (E)
  • 23 bits: Fraction (F)

Value = (-1)S × (1.F) × 2E-127

Text Representation & Other Data Types

ASCII Characters

ASCII maps 128 characters to 7-bit codes:

  • Printable characters: letters, digits, punctuation
  • Non-printable control characters: ESC, DEL, etc.

Interesting properties:

  • Difference between '0' and its ASCII code is 48
  • Uppercase and lowercase differ by 32
  • Alphabetical order matches ASCII order

Other Data Types

Common data types beyond numbers and text:

  • Text strings: Sequences of characters, often null-terminated
  • Images: Arrays of pixels (monochrome: 1 bit/pixel; color: RGB components)
  • Sound: Sequences of fixed-point numbers representing audio waveforms

These are typically implemented in software rather than directly in hardware.

LC-2/LC-3 Data Types

The LC-2/LC-3 architecture supports:

  • 16-bit two's complement signed integers
  • Operations: ADD, AND, NOT

Other data types (logical, text, fixed-point) are implemented through software interpretation of these 16-bit values.
